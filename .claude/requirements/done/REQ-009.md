# [REQ-009] 추천 친구 개선

## 개요
REQ-008에서 구현한 추천 친구 기능의 추천 범위 기준을 변경하고,
추천 카드에 공통 팔로우 수를 추가 노출한다.

## API
| Method | Path | 설명 |
|--------|------|------|
| GET | /api/v1/social/recommend | 추천 친구 목록 조회 (응답 필드 추가) |

### 응답 변경 (RecommendUserResponse)
| 필드 | 타입 | 설명 |
|------|------|------|
| userId | long | 추천 유저 ID |
| nickname | String | 닉네임 |
| profileImgNumber | int | 프로필 이미지 번호 |
| mutualFollowCount | int | 추천 유저의 팔로잉 중 내가 팔로잉하는 유저 수 |

## 비즈니스 규칙

### 1. 추천 범위 기준 변경 (±1 티어 그룹 → ±1 티어)
- **변경 전**: 같은 메인 티어 그룹 (브론즈 1~3 전체를 하나의 그룹으로 취급)
- **변경 후**: 같은 sort_order (개별 티어) 기준으로 추천
  - 기본: 나와 정확히 같은 티어(sort_order) 유저
  - 폴백: 같은 티어 유저가 5명 미만일 경우 sort_order ±1 범위로 확장
  - 예) 브론즈 2(sort_order=2)라면 → 폴백 대상은 브론즈 3(sort_order=1), 브론즈 1(sort_order=3)

### 2. 아는 사람 수 (mutualFollowCount) 응답 추가
- 내가 팔로잉하는 사람 중 이 추천 유저를 팔로잉하는 사람의 수
- 인스타그램의 "A님, B님 외 N명이 팔로우합니다"와 동일한 개념
- 화면 예시: "내이름님 외 2명" → mutualFollowCount=3일 때
- DB에서 서브쿼리로 계산하여 함께 조회
- 계산 방식: COUNT( 내가 팔로잉하는 X | X가 추천 유저를 팔로잉함 )

## 참고
- 이전 요구사항: REQ-008
- 변경 파일: RecommendUserRepositoryImpl, RecommendUserService, RecommendCandidateDto, RecommendUserResponse, SocialFacade, UserLeagueService
- sort_order가 1부터 시작하므로 폴백 범위 하한은 MAX(1, sort_order - 1)
