# [REQ-011] 소셜 알림 (팔로우 / 축하하기 받음 / 친구 활동)

## 개요
팔로우, 축하하기, 친구 성취 이벤트가 발생했을 때 즉시 알림을 발송하여 소셜 상호작용과 학습 동기를 자극한다.
(요구사항 명세 3.9, 3.10, 3.11)

## API
| Method | Path | 설명 |
|--------|------|------|
| - | - | 외부 API 없음. 도메인 이벤트(팔로우/축하/친구 활동) 발생 시 리스너 기반으로 발송 |

## 비즈니스 규칙

### 1. 팔로우 알림 (3.9) 🟢
- 발송 조건: 팔로우 이벤트 발생 즉시
- NotificationType: `FOLLOW`
- 문구: `OO님이 나를 팔로우했어요! 👀` (OO: 팔로우한 유저 닉네임, 동적 삽입)
- 액션 버튼:
  - 내가 팔로우하지 않은 유저가 팔로우한 경우 → **맞팔로우** 버튼 활성화 (`FOLLOW_BACK`, targetId = 상대 userId)
  - 내가 이미 팔로우한 유저가 팔로우한 경우 → 버튼 없음
- 액션 버튼 삽입 정책은 기획 확정 필요 (명세상 "액션 버튼 삽입에 대한 의견 구하기" 표기, 미정)

### 2. 축하하기 받은 알림 (3.10) 🟢
- 발송 조건: 친구가 내 활동 피드에서 축하하기 버튼 클릭 즉시
- NotificationType: `CONGRATULATION`
- 문구: `OO님이 축하해줬어요! 🎉` (OO: 축하한 유저 닉네임, 동적 삽입)
- 액션 버튼: 없음 (`NONE`)
- 축하하기 기능 본체: REQ-007 참고

### 3. 친구 활동 알림 (3.11) 🟢
- 발송 조건: 팔로잉한 친구의 아래 성취 이벤트 발생 즉시
- NotificationType: `FRIEND_ACTIVITY`
- 액션 버튼: 축하하기 (`CONGRATULATE`, targetId = feedId)
  - 오늘 축하하기 잔여 횟수 1회 이상 → **축하하기** 버튼 활성화
  - 오늘 축하하기 3회 모두 사용 → **축하하기** 버튼 비활성화

| 이벤트 | 문구 |
|--------|------|
| 행성(도메인) 완료 | `OO님이 N행성을 정복했어요! 🌍` |
| 연속학습 달성 (마일스톤) | `OO님이 N일 연속학습을 달성했어요! 🔥` |
| 티어 승급 | `OO님이 N티어로 승급했어요! 🎉` |
| 레벨업 | `OO님이 LV.N이 됐어요! 💪` |

- OO: 친구 닉네임, N: 해당 이벤트 값 (동적 삽입)

## 참고
- 관련 도메인 패키지: `notification`, `friend`, `social`
- 관련 클래스: NotificationEventListener, NotificationService, NotificationMessageProvider, NotificationType(FOLLOW, CONGRATULATION, FRIEND_ACTIVITY), NotificationActionType(FOLLOW_BACK, CONGRATULATE)
- 친구 활동 피드: REQ-006, 축하하기: REQ-007
- 친구 활동 이벤트는 피드 적재(REQ-006)와 동일 이벤트를 재사용할지 별도 발행할지 확인 필요 (미정)
