# 요구사항 변환 및 등록

입력: $ARGUMENTS

## Phase 1: 다음 REQ 번호 결정

1. `.claude/requirements/todo/`와 `.claude/requirements/done/`의 파일 목록을 Glob으로 확인하라
   - 패턴: `REQ-*.md`
2. 존재하는 번호 중 가장 큰 값 + 1을 다음 번호로 결정하라
   - 파일이 하나도 없으면 001부터 시작 (`REQ-001.md` 템플릿 파일 제외)
   - 번호는 세 자리로 zero-padding한다 (예: 001, 002, 010)
3. 결정된 번호를 확정하라

> 다음 Phase 조건: 다음 REQ 번호가 결정되었을 때

> Skip 조건: 없음 (필수 Phase)

## Phase 2: 자유 형식 요구사항 분석

1. $ARGUMENTS로 전달된 텍스트를 분석하라
   - $ARGUMENTS가 비어있으면 사용자에게 요구사항 내용을 입력해달라고 요청하고 중단하라
2. 아래 항목을 추출하라:
   - **제목**: 기능을 한 줄로 표현하는 명사형 제목
   - **개요**: 이 기능이 왜 필요한지 한 문장
   - **API 스펙**: HTTP 메서드, 경로, 설명 (언급이 없으면 "미정"으로 표기)
   - **비즈니스 규칙**: 조건, 제약, 예외 사항 목록
   - **참고**: 관련 엔티티명, 도메인 패키지명 (추론 가능하면 추론, 불명확하면 "미정")
3. 누락된 항목이 있으면 빈칸으로 남기지 말고 "미정"으로 표기하라

> 다음 Phase 조건: 모든 항목이 추출되었을 때

> Skip 조건: 없음 (필수 Phase)

## Phase 3: REQ 파일 생성

1. Phase 2에서 추출한 내용을 아래 규격으로 변환하라:

```
# [REQ-{번호}] {제목}

## 개요
{개요}

## API
| Method | Path | 설명 |
|--------|------|------|
| {메서드} | {경로} | {설명} |

## 비즈니스 규칙
- {규칙1}
- {규칙2}

## 참고
- 관련 엔티티: {엔티티명}
- 관련 도메인 패키지: {패키지명}
```

2. `.claude/requirements/todo/REQ-{번호}.md`로 저장하라

> 다음 Phase 조건: 파일이 저장되었을 때

> Skip 조건: 없음 (필수 Phase)

## Phase 4: index.md 업데이트

1. `.claude/requirements/index.md`를 Read로 읽어라
2. `## Todo` 테이블에 아래 형식으로 한 줄 추가하라:
   ```
   | REQ-{번호} | {제목} |
   ```

> Skip 조건: 없음 (필수 Phase)

## Phase 5: 결과 보고

1. 사용자에게 아래 내용을 보고하라:
   - 생성된 파일 경로
   - 변환된 내용 요약 (제목, API, 규칙 수)
   - "미정"으로 표기된 항목이 있으면 명시하여 보완을 권유하라
