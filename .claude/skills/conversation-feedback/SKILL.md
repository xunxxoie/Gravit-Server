# 대화 피드백

현재 컨텍스트에 남아있는 대화를 분석해, 사용자가 성장할 수 있도록 피드백을 제공한다.
도구 호출 없이 이미 컨텍스트에 있는 대화만으로 분석한다. **새 파일을 만들거나 저장하지 않는다.**

## 핵심 원칙 (모든 Phase에 적용)

1. **근거 없는 평가 금지.** 칭찬이든 지적이든 반드시 대화 속 실제 발언/행동을 근거로 든다.
   막연한 일반론("질문을 더 잘하면 좋겠어요")은 금지하고, "X를 물어볼 때 Y 맥락이 빠져 있었다"처럼 구체적으로 짚는다.
2. **아첨 금지.** 좋게 포장하려는 경향을 의식적으로 억제한다. 개선점은 솔직하게 적는다.
3. **분석 대상은 사용자(나)다.** Claude(나 자신)의 행동을 평가하는 게 아니라, 사용자의 태도·지식·소통·판단을 평가한다.

## Phase 1: 대화 범위 파악

1. 현재 컨텍스트에 남아있는 대화 전체를 처음부터 훑는다.
2. 대화가 컴팩션(요약)된 흔적이 있으면, 보고 시작에 "초반부는 요약본만 보여 일부 디테일이 누락될 수 있음"을 한 줄로 명시하라.
3. 분석할 만한 대화량이 거의 없으면(예: 인사 몇 마디) 그 사실을 알리고 중단하라.

> 다음 Phase 조건: 분석할 대화 범위가 확인되었을 때

> Skip 조건: 없음 (필수 Phase)

## Phase 2: 분석

대화를 아래 네 관점으로 분석하고, 각 항목마다 **대화 속 근거**를 함께 메모하라.

1. **모르거나 헷갈린 지점** (기술 지식 격차) — CS/코드/도구에 대해 몰랐거나 잘못 알고 있던 부분.
2. **소통·질문 방식** — 요구사항이 모호했던 순간, 더 좋은 질문이 가능했던 지점.
3. **문제 정의·의사결정** — 문제를 잘 좁혔는지, 트레이드오프 판단, 방향 설정 능력.
4. **잘한 점(강점)** — 긍정적으로 평가할 습관/접근.

> 다음 Phase 조건: 네 관점 모두 분석되고 각각 근거가 확보되었을 때

> Skip 조건: 없음 (필수 Phase)

## Phase 3: 피드백 출력

아래 구조로 보고하라. **냉정한 멘토 → 균형형 코치** 순서를 지킨다.
분량은 섹션당 핵심 2~3개 불릿, 필요하면 불릿당 한 줄 부연 정도로 간결하게.

앞 세 섹션(🔍 / 💬 / 🎯)은 **냉정한 멘토** 톤으로 — 직설적이고 근거 기반으로, 개선점에 무게를 둔다.
뒤 두 섹션(✅ / 🚀)은 **균형형 코치** 톤으로 전환 — 강점을 인정하고 동기를 부여한다.

```
## 🔍 모르거나 헷갈린 지점
- (근거 기반으로 구체적으로)

## 💬 소통·질문 방식
- (모호했던 순간, 더 나은 질문 방향)

## 🎯 문제 정의·의사결정
- (방향 설정, 트레이드오프 판단)

## ✅ 잘한 점
- (강점, 근거 포함)

## 🚀 다음 한 걸음
- (다음 세션에 시도해볼 것 한 가지 + 격려 한 줄)
```

> Skip 조건: 없음 (필수 Phase)
