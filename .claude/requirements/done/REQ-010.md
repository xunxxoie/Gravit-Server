# [REQ-010] 시즌 관련 알림 (종료 임박 / 종료+새 시즌 시작)

## 개요
시즌 종료 임박과 시즌 전환을 알려 유저의 마지막 스퍼트와 재참여를 유도한다.
(요구사항 명세 3.7, 3.8)

## API
| Method | Path | 설명 |
|--------|------|------|
| - | - | 외부 API 없음. 스케줄러(NotificationScheduler) 기반으로 발송 |

## 비즈니스 규칙

### 1. 시즌 종료 임박 알림 (3.7) 🟡
- 시즌 종료 7일 전, 3일 전 두 차례 발송
- NotificationType: `SEASON_ENDING`
- 액션 버튼: 학습하러 가기 (`GO_TO_LEARNING`, targetId 없음) — 공통

| 발송 시점 | 문구 |
|-----------|------|
| 시즌 종료 7일 전 | `시즌이 일주일 뒤 끝나요! 지금이 티어 올릴 마지막 기회예요 💪` |
| 시즌 종료 3일 전 | `시즌 종료가 3일 앞으로 다가왔어요! 마지막까지 달려봐요 🔥` |

### 2. 시즌 종료 + 새 시즌 시작 알림 (3.8) 🟡
- 발송 조건: 시즌 종료 즉시
- NotificationType: `SEASON_RESET`
- 문구: `시즌 종료! 새 시즌이 찾아왔어요. 다시 시작해봐요 💪`
- 액션 버튼: 없음 (`NONE`)
- 소프트 리셋 결과(시작 티어, 시작 LP)는 알림에 포함하지 않는다. 첫 접속 시 앱 내 팝업으로 별도 노출 (리그 기능 명세서 / REQ-003 참고)

## 참고
- 관련 도메인 패키지: `notification`, `season`
- 관련 클래스: NotificationScheduler, NotificationFacade, NotificationMessageProvider, SeasonService(getActiveSeasonEndsAt), NotificationEventListener, SeasonBatchService, SeasonRolledOverEvent, NotificationType(SEASON_ENDING, SEASON_RESET)
- 시즌 주기/소프트 리셋: REQ-002, REQ-003

### 발송 시점 결정 (확정)
- 기준: ACTIVE 시즌의 `endsAt` (각 시즌이 명확한 종료 시각 보유)
- **종료 임박(3.7)**: 순수 동적 1회성 스케줄링 대신 **매일 1회 크론 + 가드** 방식 채택
  - 매일 21:00 KST 크론에서 `endsAt` 기준 남은 일수를 계산해 7 또는 3일 전이면 발송
  - 이유: Spring `@Scheduled`는 정적 cron만 지원하고, 동적 1회성 task는 메모리 상주라 배포/재시작 시 소실 위험. 기존 NotificationScheduler 일일 크론 패턴과도 일치. 일일 조회 비용은 무시 가능
- **종료 즉시(3.8)**: 롤오버 배치(`SeasonBatchService.finalizeAndRollover`) 커밋 후 `SeasonRolledOverEvent` 발행 → `NotificationEventListener`가 AFTER_COMMIT에서 전체 푸시
